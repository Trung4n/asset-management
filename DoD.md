# Definition of Done (DoD)

Phiên bản Tiếng Việt

Tài liệu này xác định các tiêu chí bắt buộc phải hoàn thành trước khi một user story, task hoặc tính năng được xem là **Hoàn Thành**.

---

# 1. Tài Liệu

- User story được xác định rõ ràng và ghi chép đầy đủ.
- Use case scenario được định nghĩa và yêu cầu được phân tích.
- Thiết kế UI đã hoàn thành và được phê duyệt.
- Thiết kế cơ sở dữ liệu / schema đã hoàn thành nếu cần.

---

# 2. Chất Lượng Mã Nguồn

- Code tuân thủ cấu trúc dự án và quy tắc đặt tên đã thống nhất.
- Code được review và phê duyệt trước khi deploy.
- Code vượt qua unit test trước khi merge vào nhánh main.

---

# 3. Kiểm Thử

- Test plan được chuẩn bị trong quá trình phát triển.
- Manual testing được thực hiện đầy đủ cho tất cả test case liên quan.
- Feedback được gửi cho developer để sửa lỗi.

---

English version

This document defines the criteria that must be completed before any user story, task, or feature is considered **Done**.

---

# 1. Documentation

- User stories are clearly identified and documented.
- Use case scenarios are defined and requirements are analyzed.
- UI design is completed and approved.
- Database design/schema is completed if required.

---

# 2. Code Quality

- Code follows agreed project structure and naming conventions.
- Code is reviewed and approved before deployment.
- Code passes unit tests before merging into the main branch.

---

# 3. Testing

- Test plan is prepared during development.
- Manual testing is completed for all related test cases.
- All acceptance criteria are verified successfully.
- Feedback is provided to developers for fixing issues.
